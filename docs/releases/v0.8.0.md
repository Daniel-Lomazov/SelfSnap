# SelfSnap v0.8.0 Release Notes

`v0.8.0` turns scheduling into a real recurring system instead of a single daily-time model. The app can now express fast repeating captures, coarse recurring captures, and a clearer editing workflow in Settings.

## What's new

- You can now add schedules as `Every N seconds`, `minutes`, `hours`, `days`, `weeks`, `months`, or `years`.
- Each schedule is anchored to a specific start date and start time in local time.
- Settings now includes a real schedule editor with `Add`, `Save`, `Cancel`, and `Delete`.
- Existing daily schedules are carried forward into the new recurrence model automatically.

## How scheduling works now

Each schedule means: repeat from the saved start date and start time every `N` chosen units.

New schedules start from these defaults:

- `Every 1 day`
- `start date = today`
- `start time = now`

You can edit the date and time by hand if you want a custom anchor.

There are two runtime paths now:

- Fast schedules such as `seconds` and `minutes` are managed by the tray while the tray is running.
- Coarser schedules such as `hours`, `days`, `weeks`, `months`, and `years` are registered through Windows Task Scheduler.

## What changed in Settings

The old freeform schedule editing flow was replaced with a structured editor.

- Select no schedule to add a new one.
- Select one schedule to edit it in place and use `Save` or `Cancel`.
- Select multiple schedules to delete them together.

The schedule list now shows the recurrence, start point, and enabled state directly in the UI.

## Important behavior to know

- `seconds` and `minutes` schedules require the tray to be running in the logged-in session.
- `months` and `years` skip invalid dates instead of moving to the last day of the period.
- Migrating to the new schedule model changes how schedules are represented, not your existing stored captures or capture history.
